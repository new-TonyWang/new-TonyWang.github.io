Lookahead-Decoding

发表于 2024-04-14 分类于 AI Disqus：

背景

Lookahead Decoding 是一种并行解码算法，它的主要目标是加速大型语言模型（LLM）的推理过程。

在decoder only架构的传统的自回归解码中，模型会通过prefill和decode两个阶段解析prompt，并逐一生成token。传统的自回归解码是串行方式一个一个得生成新token，这会导致推理速度较慢。研究人员为了解决这个问题，也提出了speculative decoding(投机推理) 和 parallel decoding(并行推理)两种方案，其中 speculative decoding需要依赖draft模型来快速产生多个token，然后再由大模型进行校验来提升推理速度；而parallel decoding是让大模型通过雅可比迭代法求解解方程组得方式并行的输出多个token，来提升推理速度。

然而，这两种方案各有一些不足。例如，speculative decoding 需要额外训练draft模型，而parallel decoding经常导致正确的token被放在了错误的位置，进而导致模型推理的次数增加，削弱了并行解码的收益。

Lookahead Decoding 基于上述两个方案的思想，它通并行解码和n-gram词组的方式提升了模型推理速度。这里的n-gram可以理解为由使用已经生成的多个token组成的词组。 Lookahead Decoding 会将以往推测出的词组记录下来，在下次推理之前从历史词组中选取可能的推理结果，然后通过并行推理的方式校验推理的结果。

算法逻辑

算法有两个关键的阶段：

  • 预热阶段：填充past_token，准备token_map。该阶段只会运行lookahead分支，不会运行verify分支。
  • 解码阶段：从token_map中找lst_token对应的n-gram，并运行lookahead和verify分支。

在进入详细算法之前，我们需要先介绍算法中的几个关键变量：

  • WINDOW_SIZE(W)：算法经过预热之后，每一组向前lookahead的token数量。预热时的WINDOW_SIZE会稍大一些。
  • GUESS_SET_SIZE(G)：校验分支每一个token最多支持的候选词组数量，G的数量会影响校验分支的运算量，G越小校验分支运算量越小。例如G=2时，Who只能最多有两个候选词 Who->am->I和Who->are->you，其他Who->xx的候选会将前两个替代。
  • LEVEL(N)：指定了n-gram的大小，n-gram是通过之前的n次雅可比迭代得到的数值组成。例如N=7，那么就可以向前看7个token，并由最多7个token组成短语。
  • token_map: 是代码中实现n-gram池子的变量，它是一个map，key值是token，value是由该token组成的n-gram词组。例如{"who",[[am,I],[are,you]]}，value的长度由LEVEL控制。
  • past_tokens: 是代码中的关键二维数组，预热之后，它的shape是N-1行W列。预热阶段就是填充该二维数组，每一次前向推理就会填充其中一行，知道数组被填充满时推理结束。token_map也是从该数组构建。
  • guess_tokens：代码中用于表示猜测token的数组，它由多个可能的n-gram拼起来。例如Who的guess_tokens是[am,I,are,you]
  • lst_token：代码中用于表示猜测正确的词组的最后一个token。在预热阶段lst_token恒为1，因为预热阶段没有校验分支，只能通过自回归推断一个正确的token；预热结束之后，会通过校验分支校验guess_tokens而得来。
  • input_ids：代码中自投机推理的输入。

MASK矩阵设置

个人认为论文中的mask这张图内容极多，理解了mask就差不多理解整个算法。

论文中的设置为W = 5, N = 4, G = 2。 结合上文的算法，我们可以理解为，这张图已经经过了预热阶段。 整个模型的输入被分成了3个部分：

  • input_token: 就是左上角深蓝色的token，它们是已经确定的token。
  • Lookahead Branch token: 这部分token就是由past_token所有行数组拼接出的token。图中用橙色，绿色和红色标注。
  • Verification Branch token： token_map中token 0 对应的候选token，并且它有两个候选。

三部分的输入拼接起来才构成了模型的真正的输入。

通过mask矩阵，我们可以看出： 1. 所有的token都可以看到input_token； 2. Lookahead Branch token只能看到当前迭代的token，看不到其他迭代的token，也看不到Verification Branch token。 3. Verification Branch token 只能看到当前候选词对应的token。

past_tokens和自投机推理的输入在预热阶段的变化

如下图，W=5，N=2，G在预热阶段未用到。浅蓝色表示完全正确的token。图中投机推理输入的蓝色部分是一定正确的token。 在初始化的时候，初始化窗口大小为7，past_tokens先通过初始化（绿色的部分），然后绿色部分推出橙色，橙色推红色，红色推黄色。等到past_token被填满的时候，才开始新增校验分支的token（浅蓝色）。

jforward_multilevel 函数

这个函数是对llama模型前向推理的封装函数，它的作用是：

  • 将3部分的输入进行拼接，得到最终的输入，position_id和attention_mask数组。
  • 执行llama前向函数。该函数内部会生成MASK矩阵，并执行前向推理
  • 处理模型输出logist，生成outputs.out_logits，outputs.inp_logits和outputs.guess_logits。
