CUDA入门总结

本周主要学习了CUDA相关的知识，也算是对GPU入了个门，以方便后续深入学习ai编译器。

以下是本周的总结。

异构计算和CUDA

  • 左图：一个四核CPU一般有四个ALU，ALU是完成逻辑计算的核心，也是我们平时说四核八核的核，控制单元，缓存也在片上，DRAM是内存，一般不在片上，CPU通过总线访问内存。
  • 右图：GPU，绿色小方块是ALU，我们注意红色框内的部分SM，这一组ALU公用一个Control单元和Cache，这个部分相当于一个完整的多核CPU，但是不同的是ALU多了，control部分变小，可见计算能力提升了，控制能力减弱了，所以对于控制（逻辑）复杂的程序，一个GPU的SM是没办法和CPU比较的，但是对了逻辑简单，数据量大的任务，GPU非常适合。

就以我的3060来说，具体配置如下：

3060是Ampere架构，一共3584个CUDA核心，最多支持threadBlock的上限是(1024,1064,64)，grid数量是(2147483647, 65535, 65535)，由32个线程组成一个warp，常量内存64KB，block的共享内存48KB，block寄存器65536个。

在已知上面的信息之后，我们就可以开始编写cuda程序了。

CUDA编程模型

一个异构环境，通常有多个CPU多个GPU，他们都通过PCIe总线相互通信，也是通过PCIe总线分隔开的。所以我们要区分一下两种设备的内存：

  • 主机：CPU及其内存
  • 设备：GPU及其内存

我们从内存，线程，核函数三个层面来了解CUDA

内存管理

上文提及CUDA程序有host内存和device内存，对于device内存，需要调用CUDA相关的API进行管理。下表给出了host 的api和device api的对比

标准C接口 CUDA C接口 功能
malloc cudaMalloc 内存分配
memcpy cudaMemcpy 内存复制
memset cudaMemset 内存设置
free cudaFree 释放内存

在host端可以指定device侧的内存分配与释放，device侧的内存也是有层级关系的，从快到慢依次是block中的寄存器，block内部共享的shared memory和所有grid共享的global memory。

需要注意的是，一般情况下cuda kernel只能访问device侧的数据，访问host内存时会失败，但是程序并不会因此退出，类似的，host 侧代码也不能访问device的内存，否则会出现段错误。

线程管理

在GPU中，一个核函数只能有一个grid，一个grid可以有很多个Block，每个Block可以有很多的Thread，这种分层的组织结构使得我们的并行过程更加自如灵活，注意不同块内线程不能相互影响，他们是物理隔离的。

上图中我们观察到，所有的Block和Thread都是有一个坐标来唯一标识的，这种标识方法是CUDA独有的一套机制。CUDA使用dim3类型结构体来标识Block和Thread.

dim3就是由x,y,z构成的结构体，在host端执行kernel前需要指定该kernel使用的grid和block的dim3结构体，device侧需要获取当前block和thread的标识并绑定到具体运算。

如上述代码所示，host侧的写法和常见的C++类似，但是在kernel函数中就有所不同。在kernel函数中依靠下面两个内置结构体确定线程标号：

  • blockIdx（线程块在线程网格内的位置索引）
  • threadIdx（线程在线程块内的位置索引）

这两个内置结构体基于 uint3 定义，包含三个无符号整数的结构，通过三个字段来指定：

  • blockIdx.x
  • blockIdx.y
  • blockIdx.z
  • threadIdx.x
  • threadIdx.y
  • threadIdx.z

上面这两个是坐标，当然我们要有同样对应的两个结构体来保存其范围，也就是blockIdx中三个字段的范围threadIdx中三个字段的范围：

  • blockDim
  • gridDim

他们是dim3类型(基于uint3定义的数据结构)的变量，也包含三个字段x,y,z.

  • blockDim.x
  • blockDim.y
  • blockDim.z

网格和块的维度一般是二维和三维的，也就是说一个网格通常被分成二维的块，而每个块常被分成三维的线程。 注意：dim3是手工定义的，主机端可见。uint3是设备端在执行的时候可见的，不可以在核函数运行时修改，初始化完成后uint3值就不变了。他们是有区别的！这一点必须要注意。

在C语言函数前没有的限定符global ,CUDA C中还有一些其他我们在C中没有的限定符，如下：

限定符 执行 调用 备注
__global__ 设备端执行 可以从主机调用也可以从计算能力3以上的设备调用 必须有一个void的返回类型
__device__ 设备端执行 设备端调用
__host__ 主机端执行 主机调用 可以省略
